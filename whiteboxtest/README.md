# 🌲 Whitebox Workflows Building Mapping & Tree Filtering

This directory contains scripts and test workflows for building footprint extraction from raw LiDAR datasets using **Whitebox Workflows for Python (WbW)**. 

Specifically, this directory documents how to run building extraction pipelines without a premium **WbW-Pro** license, and how to filter out vegetation and trees from raw point clouds.

---

## 🛠️ Scripts in this Directory

*   **[`wbwbuildingmapping.py`](file:///Users/liammacleod/Nextcloud/MASC/BuildingMapGIS/whiteboxtest/wbwbuildingmapping.py)**: The main building footprint extraction script. It processes raw LiDAR files, performs DEM interpolation, extracts off-terrain objects (OTOs), filters features by height/area, and smooths the resulting vector polygons.
*   **[`view_outputs.py`](file:///Users/liammacleod/Nextcloud/MASC/BuildingMapGIS/whiteboxtest/view_outputs.py)**: A visualization utility using `rasterio`, `geopandas`, and `matplotlib` that generates a premium three-panel plot comparing the interpolated DEM, building raster mask, and final vector polygon footprints.
*   **`output/`**: The output directory containing:
    *   `DEM.tif`: The interpolated Digital Elevation Model (excluding vegetation and noise).
    *   `buildings.tif`: The final boolean building raster mask.
    *   `building_footprints.*`: The shapefile containing the final building footprints vector geometries.
    *   `visualization.png`: The output comparison plot generated by `view_outputs.py`.

## 💡 Key Learnings & Implementations

### 1. Bypassing WbW-Pro Licensing (Area Filtering Workaround)
In standard WbW scripts, small objects (like noise or single trees) are filtered out using the premium `generalize_classified_raster` tool. 

To run this workflow **without** a WbW-Pro license, we designed a workaround combining standard WbW tools:
1.  **`clump`**: Group adjacent positive building cells into unique spatial patches (clumps).
2.  **`zonal_statistics`**: Count the number of cells within each clump using `stat_type="total"`.
3.  **Raster comparison**: Retain only clumps that meet or exceed the `min_area` threshold (in grid cells).

```python
# License-free clump-based area filter
otos_clumped = wbe.clump(otos, zero_background=True)
clump_sizes, _ = wbe.zonal_statistics(otos, otos_clumped, stat_type='total', zero_is_background=True)
otos = clump_sizes >= min_area
```

### 2. Advanced Tree & Vegetation Filtering
In raw point clouds, trees and vegetation often pass height and area thresholds, causing them to be falsely detected as buildings. We investigated two ways to filter them out using point classification:

#### Method A: LiDAR Point Exclusions during DEM Interpolation (Recommended)
By analyzing the class distribution of our raw McMaster LiDAR dataset (`ON_Niagara_20210525_NAD83CSRS_UTM17N_1km_E587_N4790_CLASS_standard.laz`), we discovered that **42.09%** of the points belong to **Class 1 (Unclassified/Vegetation)**.

If Class 1 points are not excluded, they are interpolated into the DEM as high peaks and processed as off-terrain building candidates. By passing Class 1 along with standard vegetation/noise classes to the `excluded_classes` parameter of `lidar_tin_gridding`, we construct a DEM consisting of *only* ground and actual building roofs:

```python
# Exclude Class 1 (Trees/Unclassified), Class 3,4,5 (Veg), Class 7 (Low Noise), and Class 17,18 (Bridge/High Noise)
dem = wbe.lidar_tin_gridding(
    lidar, 
    returns_included='last', 
    cell_size=resolution, 
    excluded_classes=[1, 3, 4, 5, 7, 17, 18]
)
```
*   **Result:** False-positive footprint count dropped from **388** down to **273** (a **30%** reduction in noise, leaving only actual buildings).

#### Method B: Direct Class 6 (Building) Gridding
Alternatively, if the input point cloud is already classified, we can grid *only* Class 6 (Building) points. To prevent triangulation triangles from bridging gaps between buildings, we limit the maximum triangle edge length:

```python
dem_building = wbe.lidar_tin_gridding(
    lidar, 
    returns_included='all', 
    cell_size=resolution, 
    excluded_classes=[1, 2, 3, 4, 5, 7, 17, 18],
    max_triangle_edge_length=5.0
)
```
*   **Result:** Highly strict extraction (80 footprints), but can miss smaller details if building points are sparse or spaced far apart.
